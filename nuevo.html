Curso de Git desde cero

Como Subir un repositorio

  1. git init

    2. crea un nuevo repositorio de Git en tu directorio actual.

  2. git add .

    3. El comando git add prepara los archivos modificados o nuevos moviéndolos al área de preparación (staging area). Esto le indica a Git qué cambios formarán parte de la próxima confirmación

  3. git commit -m "Mensaje"

    4. El comando git commit -m "Mensaje" guarda en tu historial los archivos preparados previamente. La letra -m indica que escribirás un mensaje entre comillas, el cual debe describir brevemente los cambios realizados (por ejemplo: git commit -m "Corrección: fix en el menú de navegación"

  4. git branch -M main

  5. git branch -M main

    6. Una Git branch es una especie de copia del código en un punto concreto del proyecto. Es como tener varias versiones del mismo trabajo a la vez, pero separadas. Así, puedes realizar cambios en una rama sin miedo a romper nada.

  6. git remote add origin https://github.com/

    7. El comando git remote add origin vincula tu repositorio local a un servidor remoto (como GitHub o GitLab). La palabra "origin" es solo el nombre predeterminado que se le da a esa URL

  7. git push -u origin main

    8. Este comando envía los cambios de la rama local en la que te encuentras actualmente a la rama remota con el mismo nombre.
